Some CMS API Documentation

The API is delivered as a simple web service. Queries use GET and updates use POST. The API is organized into sections and actions. URLs for API methods are formed as follows: some/api/[section]/[action]. If action is omitted, the index action is assumed.

Page

The /page section contains actions to manage pages within the CMS.

List

some/api/page/list

Returns a list of pages.

Options:
limit
A numeric value specifies how many pages to return. The first 200 are returned if not specified. A range can also be specified in the form offset,limit such that 10,10 returns pages 11 through 20.
